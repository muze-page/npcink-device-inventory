# WordPress.org Release Audit

Date: 2026-06-21

This document summarizes the WordPress.org plugin directory preparation work for
Npcink Device Inventory.

## Current Release Package

- Plugin slug: `npcink-device-inventory`
- Plugin version: `2.6.1083`
- Release package: `release/npcink-device-inventory.zip`
- Expected zip root directory: `npcink-device-inventory/`

The package script was updated to generate the WordPress.org-style filename
`npcink-device-inventory.zip`. The previous custom filename
`npcink-device-inventory-plugin.zip` is removed during packaging to avoid uploading
the wrong artifact.

  - `Requires at least`

- Added screenshot captions for WordPress.org plugin directory assets.

The package now includes:

`README.txt` now documents that the bundled React assets can be rebuilt by

### GPL, Trademark, and Asset Risk

Removed bundled image assets that had unclear licensing or trademark risk:

- Platform images
- Payment images
- OS/device images
- Unused SVG table icons
- Default Vite icon

The UI now uses text labels instead of those bundled image assets.

Brand-specific example text and links were also removed from packaged source:

- Platform defaults were changed to generic labels such as `电商平台A`.
- Payment defaults were changed to generic labels such as `电子钱包A`.
- Demo product links were changed to `https://example.com/...`.
- Hardcoded `localhost:10048` examples were removed from shipped source.
- Vite dev proxy now uses `WP_DEV_PROXY_TARGET` instead of a hardcoded local URL.

### Direct File Access Protection

Added direct access guards to admin partial PHP files:

```php
if (!defined('ABSPATH')) {
    exit;
}
```

### PHP and WordPress Coding Fixes

- Replaced `mt_rand()` with `wp_rand()`.
- Reworked CSV generation to avoid `fopen('php://temp')`.
- Removed the active use of `load_plugin_textdomain()` because WordPress.org can
  auto-load translations for the plugin text domain.
- Added gettext coverage for the admin menu title, loading text, plugin action
  link, public search page title, and public search fallback text.
- Added a locale-aware `dataLocal.labels` map for the bundled admin React
  top-level tabs.
- Regenerated `languages/npcink-device-inventory.pot`.
- Renamed global bootstrap/uninstall functions with the plugin prefix.
- Added scoped PHPCS comments for reviewed custom-table exceptions:
  - REST handlers read and mutate plugin-owned inventory tables.
  - Activation routines create/update plugin-owned custom tables, indexes, and triggers.
  - Uninstall cleanup checks and drops only plugin-owned custom tables.
  - Schema introspection uses `INFORMATION_SCHEMA` with request-scoped static caches.

## Packaging Guardrails

`.github/scripts/package-wordpress-plugin.sh` now verifies:

- `README.txt` exists and is not empty.
- Plugin version and readme Stable tag match.
- Forbidden development paths are not included.
- `node_modules`, `.env`, `.DS_Store`, and sourcemaps are not included.
- Previously removed platform/payment image files do not reappear.
- Known demo secrets and wildcard CORS patterns do not appear in the package.

The package intentionally includes front-end source and build config, but not
`node_modules`.

## CI Guardrails

The preview and release workflows now build and upload the same WordPress.org
package filename generated by the packaging script:

- `release/npcink-device-inventory.zip`

The WordPress plugin jobs also run the local PHP gates before packaging:

- `composer phpcs`
- `composer phpstan`

After packaging, the workflows run the official Plugin Check action against the
generated package directory:

- `wordpress/plugin-check-action@v1`
- `build-dir: ./release/npcink-device-inventory`

## Remaining Notes

- The plugin still uses custom database tables. This is acceptable for this
  product, but the code should keep the current allow-listing, preparation, and
  scoped PHPCS explanations intact.
- Existing installations may still contain old platform or payment values in
  their database. The code preserves existing values as returned options; only
  new defaults and packaged source were changed to generic labels.
- Before submission, upload `release/npcink-device-inventory.zip`, not the release
  directory itself.
