# 封闭试用验证记录（2026-07-15）

## 结论

当前版本已经具备进入小规模、多设备封闭试用的基础。插件安装及单台真实 macOS 设备上的源码采集器运行、首次上传、重复上传、令牌控制、后台日常操作和跨站恢复均已通过。

本轮发现并修复一个 P0：全新 WordPress 使用默认朴素固定链接时，采集器将裸站点地址拼成 `/wp-json/...`，请求会被重定向到前台页面并误判为成功。采集器现在对裸站点地址使用 `?rest_route=...`，同时拒绝把 2xx 非 JSON 页面当作成功响应。

这不是 7 天、多设备试用的完成记录。Windows、其他 macOS 机型、网络切换、重启和真实硬件变化仍需继续验证。

## 本轮误报及排除

- Docker WP-CLI 临时包装器最初没有把环境变量传入 CLI 容器，导致测试清理失败；补齐测试环境变量透传后，项目原有清理逻辑通过。
- 跨站测试最初把导出响应的 `{backup, meta}` 包装层作为备份提交，正确触发 422；改为传递 `data.backup` 后 dry-run 和恢复通过。

这两项属于测试工具使用错误，不计入产品缺陷。
