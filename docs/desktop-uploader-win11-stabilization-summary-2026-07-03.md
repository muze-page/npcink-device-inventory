# Desktop Uploader Win11 Stabilization Summary

日期：2026-07-03

本文记录 `ele-rs/` 桌面上传软件在 Windows 11 样机上的兼容性、排障能力和界面收口过程。本文不是正式发布说明；正式发布时再按发布流程打包。

## 背景

桌面上传软件最初主要在 macOS 开发和验证。Windows 版本需要确认以下能力是否可用：

- 硬件信息采集：系统、CPU、内存、硬盘、主板、BIOS、显卡、网卡。
- 运行状态监控：CPU、内存、磁盘、网络基础状态，以及可用时的高级传感器信息。
- 故障信息收集：系统事件、应用崩溃、驱动、磁盘健康、dump 和 Windows 诊断线索。
- 普通用户体验：中文界面、无明显命令行闪烁、生成排障包时有进度反馈、用户能理解软件用途和隐私边界。

## 已完成的主要调整

### Windows 打包与运行体验

- 修复 Windows 安装包中文显示问题。
- 避免启动和采集时频繁出现可见 shell 窗口。
- 排障包生成改为后台任务并通过进度条提示，降低“软件卡住”的感知。
- 修复生成排障包期间窗口短时无法拖动的问题。
- 通过 GitHub Actions `Build preview packages` 生成 Windows 预览安装包。

### 硬件与运行数据

- 补齐 Windows 上主板、BIOS、显卡等设备详情采集。
- 概览页移除多余设备编号，保留更有排查价值的硬件字段。
- 调整概览布局，让系统、处理器、显卡等长文本有更大的展示空间。
- 运行监控保留基础 CPU、内存、磁盘、网络数据；高级温度/功耗数据按平台能力显示。

### 深度排障包

排障功能最终只保留深度模式，减少用户困惑。排障包现包含：

- 应用日志、脱敏配置、权限信息和摘要。
- 静态硬件快照和运行状态快照。
- 运行历史、运行历史覆盖质量说明。
- Windows 系统、应用、安装、更新和可靠性事件。
- BIOS、主板、内存、显卡、硬盘、驱动和异常设备信息。
- 磁盘可靠性、SMART、卷、分区、电源状态。
- 网络配置、路由、连接和代理状态。
- CPU/内存占用最高进程。
- Minidump 复制与 WinDbg/cdb 自动分析；未安装调试器时写入安装提示。
- CBS、DISM 和 WER 摘要，用于系统组件损坏、更新失败和应用崩溃初筛。

隐私处理也已加强：

- `runtime-status.json` 和 `runtime-history.json` 统一走 JSON 脱敏。
- 文本文件中常见 MAC 地址格式会脱敏。
- token、主机名、序列号、UUID、用户路径等敏感信息继续脱敏。
- 命令输出中的 NUL 字节会被清理，避免文本文件被识别成二进制。

## 已验证的 Windows 排障包

最新复核包：

```text
release-preview/windows-28638578767/NpcinkDiagnostics-20260703-143945.zip
```

复核结论：

- 新增的 `cbs-log-tail.txt`、`cbs-recent-errors.txt`、`dism-log-tail.txt`、`wer-report-summary.txt`、`runtime-history-coverage.json` 已进入排障包。
- `runtime-status.json` 中 MAC 已脱敏为 `[redacted]`。
- 全包未发现明显 MAC、上传 token 或典型 Windows 主机名残留。
- `partitions.txt` 不再被识别成二进制文件。
- 摘要包含 `cbs_repair_line_count`、`minidump_count`、`wer_report_count`。
- 当前样机没有 Minidump/MEMORY.DMP，因此蓝屏类问题无法做内核 dump 分析；这是样机状态，不是采集功能失败。
- 运行历史覆盖约 31 秒，摘要中标记为 `short`。这符合快速生成排障包的场景；排查偶发卡顿时应让软件运行 5-15 分钟后再生成包。

## 界面收口

设置页已做以下收口：

- 手动填写上传配置改为弹窗，减少主界面复杂度。
- 上传配置状态移动到上传备注标题右侧。
- 软件用途说明放在设置页底部，明确说明不会读取聊天内容、屏幕画面、浏览器内容、个人文件内容，也不会录音录像。
- 软件说明文字层级已调高，保持可读，但不抢主操作区。
- 运行页长说明收进 `i` 提示，减少页面解释文字。

## 当前判断

上传软件已经达到当前阶段的目标：

- 能在 Windows 11 上安装、打开、采集、上传和生成排障包。
- 常规资产采集和远程初筛排障信息足够使用。
- 用户界面能解释软件用途，降低“监控聊天或屏幕”的误会。
- 排障包隐私边界基本可控。

当前不建议继续增加新能力。后续只处理真实使用中暴露的问题。

## 后续发布建议

正式发布前执行：

1. 在 GitHub Actions 重新生成正式 Windows 安装包。
2. 在 Windows 11 样机上安装运行。
3. 生成一次排障包并复核摘要、脱敏和新增文件。
4. 如无明显问题，再进入正式发布流程。

当前阶段不需要本地再打包，也不需要继续扩展排障范围。
