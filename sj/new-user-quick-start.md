# Npcink Device Manage 新用户快速开始

日期：2026-06-21

这份文档面向第一次安装 `Npcink Device Manage` 的站点管理员，目标是在 10 到 15 分钟内完成插件启用、基础设置、设备录入和授权上传准备。

## 1. 安装插件

1. 登录 WordPress 后台。
2. 打开 `Plugins > Add New Plugin > Upload Plugin`。
3. 上传 `npcink-device-manage.zip`。
4. 启用插件。
5. 打开 `Plugins > Device Asset Management`。

启用后插件会创建自己的数据表和默认设置。除非管理员在卸载设置中明确启用删除数据，卸载时不会自动删除资产数据。

## 2. 先完成基础设置

打开后台的 `Settings` 标签页，建议先确认：

- 折旧月限：默认 36 个月。
- 残值率：默认 5%。
- 部门列表：根据公司部门增删。
- 公共查询页面路由：默认 `public-search-page`。
- 客户端授权码：用于桌面上传软件和授权查询。

建议每个用途单独创建一个客户端授权码，例如：

- `office-upload`
- `it-public-search`
- `test-device-agent`

不用的授权码应及时禁用或删除。

## 3. 手动录入第一台设备

1. 打开 `Computer Devices` 标签页。
2. 新建设备或导入已有设备表。
3. 填写设备编号、名称、部门、状态、购买金额等字段。
4. 保存后检查列表和详情页。

如果你还没有真实设备数据，可以先录入一台测试设备，确认后台列表、筛选、导出流程都正常。

## 4. 使用桌面上传软件

桌面上传软件是可选配套工具，不包含在 WordPress 插件 zip 中。

管理员需要先准备：

1. WordPress 站点地址。
2. v2 上传接口地址：

```text
https://example.com/wp-json/npcink/v1/device-post-data-v2
```

3. 后台生成的客户端授权码。

用户在桌面端填写上传地址、授权码和可选备注后提交。提交成功后，管理员可在后台 `Computer Devices` 中看到采集到的设备记录。同一台设备再次上传时，服务端会按稳定设备 ID 更新原记录，避免重复建档。

## 5. 开启公共查询页

插件默认公共查询路由为：

```text
public-search-page
```

对应访问地址通常是：

```text
https://example.com/public-search-page/
```

公共查询页不是开放匿名数据接口。查询和上传接口都要求客户端授权码与 HMAC 签名。建议只把授权码提供给内部可信系统或人员。

## 6. 导入和导出

在 `Settings` 标签页中使用导入导出工具：

- 导出：用于备份插件自有表数据，或迁移到测试环境。
- 导入：用于恢复此前导出的插件数据。

导入前建议先备份数据库。不要把来自不可信来源的导出文件导入生产站点。

## 7. 日常维护建议

- 定期导出资产数据备份。
- 离职、设备报废或授权泄露时，及时禁用相关客户端授权码。
- 对公共查询页使用 HTTPS。
- 只给可信管理员 `manage_options` 权限。
- 升级插件前先在测试站点验证核心流程。

## 8. 常见问题

### 插件会把设备数据发给第三方吗？

不会。插件正常运行时只把数据存储在当前 WordPress 站点数据库中。

### 没有桌面上传软件还能用吗？

可以。插件后台支持手动录入、自定义设备管理、导入导出和变更记录。

### 公共查询页可以公开给所有访客吗？

不建议。公共查询和上传接口都设计为授权场景，应使用客户端授权码和签名请求。

### 卸载会删除数据吗？

默认不会。只有管理员在卸载设置中启用删除数据后，卸载才会删除插件自有表和设置。

## 9. 首次上线检查清单

- 插件已启用。
- 后台页面可以正常打开。
- 至少生成一个客户端授权码。
- 未签名上传请求返回 `403`。
- 公共查询路由符合站点实际 URL 规划。
- 已记录管理员内部的数据采集和隐私告知口径。
